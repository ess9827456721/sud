# Как выпустить новую версию «Судебного Трекера»

Автообновление работает через GitHub Releases: приложение у пользователя
сравнивает свою версию с последним релизом и тихо скачивает обновление.

## Важно: репозиторий должен быть публичным

Для обновления **без токена у пользователей** репозиторий (или отдельный
releases-репозиторий) должен быть публичным. Если репозиторий приватный:

- вариант А — создать публичный репозиторий только под релизы и указать его
  в `electron/package.json` → `build.publish`;
- вариант Б — раздавать обновления с собственного HTTPS-сервера
  (`provider: "generic"`).

**Не встраивайте GitHub-токен в приложение.**

## Шаги выпуска

1. **Поднимите версию** в `electron/package.json` (`"version": "2.1.1"` и т.п.).
   Соблюдайте semver: патч — исправления, минор — новые функции.

2. **Получите токен** (нужен только разработчику для загрузки релиза):
   https://github.com/settings/tokens → scope `repo`.

   ```bat
   set GH_TOKEN=ghp_ваш_токен
   ```

3. **Запустите публикацию** из корня репозитория:

   ```bat
   publish_release.bat
   ```

   Скрипт: собирает Python-ядро (PyInstaller + Chromium), затем
   `electron-builder --publish always` — создаёт GitHub Release и загружает
   артефакты.

4. **Проверьте релиз** на странице
   https://github.com/ess9827456721/sud/releases — в нём должны быть:
   - `Судебный Трекер Setup X.Y.Z.exe`
   - `latest.yml` (без него автообновление не работает!)
   - `.blockmap` (дифференциальные обновления)

## Как обновление приходит пользователю

- при запуске приложение тихо проверяет обновления (через 10 секунд после
  открытия окна); сбои сети игнорируются;
- если версия новее — скачивается в фоне, затем диалог
  «Обновление готово. Перезапустить сейчас?»;
- по подтверждению приложение закрывается (включая Python-процесс),
  инсталлятор ставит новую версию и запускает её;
- база данных и настройки в `%APPDATA%\SudTracker` не затрагиваются;
- ручная проверка: меню трея или Настройки → «Проверить обновления»;
- отключение автопроверки: Настройки → чекбокс «Автоматически проверять
  обновления» (хранится в `%APPDATA%\SudTracker\settings.json`).

## Подпись кода

Сборка не подписана: при первой установке Windows SmartScreen может показать
предупреждение. Сам механизм electron-updater с неподписанными NSIS-сборками
работает.
