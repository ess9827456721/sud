# Как выпустить новую версию «Судебного Трекера»

Автообновление работает через GitHub Releases: приложение у пользователя
сравнивает свою версию с последним релизом и тихо скачивает обновление.

## Основной способ — GitHub Actions (в облаке)

Локальная сборка и личный токен не нужны.

1. **Поднимите версию** в `electron/package.json`
   (`"version": "2.1.2"` и т.п., semver: патч — исправления, минор — функции).
2. **Слейте изменения** в основную ветку.
3. На GitHub откройте **Actions → workflow «Release» → Run workflow**
   (либо запушьте тег `vX.Y.Z`).
4. Через 15–25 минут проверьте страницу
   https://github.com/ess9827456721/sud/releases — релиз должен быть
   **опубликован (не Draft)** и содержать:
   - `Судебный Трекер Setup X.Y.Z.exe`
   - `latest.yml` (без него автообновление не работает!)
   - `.blockmap` (дифференциальные обновления)

   Проверяйте в режиме инкогнито/без логина — так видно то же, что видит
   updater у пользователя.

`releaseType: "release"` в `electron/package.json` обязателен: значение по
умолчанию — `draft`, а черновики релизов невидимы для electron-updater.

## Резервный способ — локальная сборка

```bat
set GH_TOKEN=ghp_ваш_токен   (scope: repo, нужен только разработчику)
publish_release.bat
```

Скрипт собирает Python-ядро (PyInstaller + Chromium) и публикует релиз
через `electron-builder --publish always`.

## Важно: репозиторий должен быть публичным

Для обновления **без токена у пользователей** репозиторий (или отдельный
releases-репозиторий) должен быть публичным. Если репозиторий приватный:
публичный releases-only репозиторий либо собственный HTTPS-сервер
(`provider: "generic"`). **Не встраивайте GitHub-токен в приложение.**

## Как обновление приходит пользователю

- при запуске приложение тихо проверяет обновления (через 10 секунд после
  открытия окна); сбои сети пишутся только в журнал;
- если версия новее — скачивается в фоне, затем диалог
  «Обновление готово. Перезапустить сейчас?»;
- по подтверждению приложение закрывается (включая Python-процесс),
  инсталлятор ставит новую версию и запускает её;
- база данных и настройки в `%APPDATA%\SudTracker` не затрагиваются;
- ручная проверка: меню «Файл → Проверить обновления», трей или Настройки;
- журнал обновлятора: `%APPDATA%\SudTracker\logs\updater.log`.

## Синхронизация сборочных файлов

Флаги PyInstaller в `.github/workflows/release.yml` — зеркало
`electron/build_electron.bat`. Меняете один файл — обновите второй
в том же коммите (в обоих стоят перекрёстные пометки).

## Подпись кода

Сборка не подписана: при первой установке Windows SmartScreen может показать
предупреждение. Сам механизм electron-updater с неподписанными NSIS-сборками
работает.
