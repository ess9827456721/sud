# Судебный Трекер — заметки для разработки

## Правила для Windows .bat/.cmd файлов (ОБЯЗАТЕЛЬНО)

Все `.bat`/`.cmd` файлы в репозитории:

- **только чистый ASCII** — сообщения на английском, никакой кириллицы;
- **переводы строк CRLF** (закреплено в `.gitattributes`);
- внутри блоков в скобках `if (...) / for (...)`: литеральные скобки в `echo`
  экранировать как `^(` и `^)`;
- комментарии — `rem`, а не `::` (внутри скобочных блоков `::` ломает парсер).

Причина: cmd.exe читает bat-файлы побайтово в OEM-кодировке (cp866).
UTF-8 с кириллицей и LF-переводами сдвигает парсер — появляются ошибки вида
«'errorlevel' is not recognized…». Неэкранированная `)` внутри `echo` в
блоке `if` досрочно закрывает блок и молча обрывает скрипт.

## Сборка релиза

`electron/build_electron.bat` и `.github/workflows/release.yml` содержат
ОДИН И ТОТ ЖЕ набор флагов PyInstaller — при изменении одного файла обновляй
второй в том же коммите (в обоих стоят перекрёстные пометки).

Публикация: GitHub Actions → workflow «Release» (см. RELEASING.md).
`publish_release.bat` — офлайн-резерв.
